# Boss招聘助手 - 产品说明文档

## 一、项目背景与目标

Boss主要从事陪诊服务业务，常年需要在 Boss 直聘平台上招聘陪诊员。当前招聘流程依赖 HR 人工操作：筛选候选人、发送招呼、跟进回复、收集简历，重复性强、效率低。

本工具为Boss HR 量身定做，运行于 Boss 直聘网页端（Tampermonkey 用户脚本），通过 AI 驱动的自动化流程，显著提升招聘效率。

---

## 二、技术架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 运行环境 | Tampermonkey | 浏览器插件，直接运行于 Boss 直聘页面 |
| 脚本语言 | 原生 JavaScript | 无需额外框架依赖 |
| AI 接入 | Dify Chat App | 自部署 Dify 平台，对话式 AI |
| 数据存储 | Tampermonkey GM_* API | 本地持久化（候选人数据、配置、已处理记录） |
| 浏览器自动化 | DOM 操作 + GM_xmlhttpRequest | 页面元素交互 + Dify API 调用 |

## 三、核心功能

### 3.1 候选人收集（推荐候选人页面）

**触发**：HR 手动进入 Boss 直聘「推荐候选人」页面，点击面板「开始收集」。

**执行流程**：

1. **按城市筛选岗位**：根据「目标城市」配置，勾选对应城市标签
2. **过滤离职人员**：排除在职候选人，聚焦有求职意向的人群
3. **收集候选人卡片**：滚动页面加载更多，从 DOM 提取候选人信息（ID、姓名、职位）
4. **陪诊岗位过滤**：只保留职位名称或候选人姓名中包含「陪诊」二字的候选人，跳过其余
5. **自动打招呼**：点击「立即沟通」按钮，对每位候选人发送招呼
6. **计数保护**：达到「招呼上限」后自动停止（默认 30 人）

**数据持久化**：

- 将已招呼候选人的 ID 存入 `processedCandidateIds`（Set），防止重复招呼
- 候选人信息存入 `candidates` 数组，含收集时间、城市、招呼状态等字段

---

### 3.2 自动沟通（消息页面）

**触发**：HR 手动进入 Boss 直聘「消息」页面，点击面板「开始沟通」。

**执行流程**：

```
进入消息列表
    │
    ▼
遍历所有候选人对话
    │
    ▼
陪诊过滤（职位/姓名含"陪诊"）
    │                              ── 不符合 ──→ 跳过，继续下一个
    ▼ 符合
城市过滤（职位字段含目标城市）
    │                              ── 不符合 ──→ 跳过，继续下一个
    ▼ 符合
时间范围过滤（超出时间范围则跳过）
    │                              ── 不符合 ──→ 跳过，继续下一个
    ▼ 符合
点击进入聊天详情
    │
    ▼
持续监听 HR 新消息 Badge（每 3 秒轮询）
    │
    ├─ 未检测到新消息 → 继续监听
    │
    └─ 检测到新消息（Badge 出现）
            │
            ▼
        识别 HR 发送的「Boss 直聘招呼语」
        （"你好，在考虑新的工作机会吗？我是Boss的人事..."）
            │
            ├─ 不匹配 → 忽略，继续监听
            │
            └─ 匹配成功
                    │
                    ▼
                HR 发送「陪诊要求」（来自配置）
                    │
                    ▼
                HR 发送「请问方便兼职么？」（固定话术）
                    │
                    ▼
                等待候选人回复（3 秒）
                    │
                    ▼
                获取候选人最新一条消息
                    │
                    ▼
                ── 无回复 → 继续监听
                │
                └─ 有回复
                        │
                        ▼
                    调用 Dify 对话 Agent
                    传入：候选人回复 + 陪诊要求 + 订单类型 + 是否指定医院
                        │
                        ▼
                    解析 AI 回复
                    从回复末尾提取标签（true/false）
                    去除标签后获得纯净消息正文
                        │
                        ▼
                    发送 AI 消息正文给候选人
                        │
                        ├─ 标签为 false → 候选人不合格，跳出该候选人对话
                        │
                        ├─ 标签为 true  → 候选人合格，收藏 + 求简历，跳出
                        │
                        └─ 无标签       → 对话继续，监听候选人下一条回复
```

**Dify 对话 Agent 标签协议**：

| AI 回复末尾标签 | 含义 | 脚本行为 |
|---------------|------|---------|
| `...消息内容 true` | 候选人符合要求 | 发送消息后收藏 + 求简历（break） |
| `...消息内容 false` | 候选人不符合要求 | 发送消息后跳过（break） |
| `...消息内容 continue` | 对话继续，等待候选人下一条回复 | 继续监听 badge 变化 |
| `...消息内容`（无标签） | 同 continue，对话继续 | 继续监听 badge 变化 |

> **注意**：标签大小写不敏感，`True` / `TRUE` / `true` 均能正确识别。

## 四、配置字段说明

所有配置通过面板 UI 填写，数据存储在浏览器本地（Tampermonkey GM Storage）。

| 字段名 | 变量名 | 类型 | 默认值 | 说明 |
|--------|--------|------|--------|------|
| 目标城市 | `targetCity` | 文本 | （空） | 招聘岗位所在城市，用于筛选候选人 |
| 招呼上限 | `greetLimit` | 数字 | 30 | 自动停止前最多发送招呼的人数 |
| 时间范围 | `timeRange` | 单选 | 全部 | 沟通记录的时间范围过滤（全部/今日/近3日/近1周/近2周） |
| 陪诊要求 | `escortRequirements` | 多行文本 | （空） | 本次陪诊任务的具体信息（就诊日期、城市、医院等），发送给 Dify AI 生成个性化消息 |
| 订单类型 | `orderType` | 单选 | 普通单 | 「急单」或「普通单」，影响 AI 生成内容 |
| 本单是否指定固定医院 | `hospitalRequired` | 单选 | 否 | 「是」或「否」，影响 AI 生成内容 |

## 五、Dify Agent 接口规范

### 5.1 对话 Agent（Chat App）

- **地址**：`POST http://192.168.31.20:6080/v1/chat-messages`
- **鉴权**：`Authorization: Bearer app-cZouV7tO88rbNEncjCjIWiDj`
- **请求体**：
  ```json
  {
    "query": "<候选人最新回复消息>",
    "inputs": {
      "info": "<陪诊要求（来自配置）>",
      "orderType": "<订单类型（来自配置）>",
      "hospitalRequired": "<是否指定医院（来自配置）>"
    },
    "response_mode": "blocking",
    "user": "hengyu-assistant"
  }
  ```
- **响应**：`{ "answer": "AI回复内容（含末尾标签）" }`
- **超时**：60 秒

### 5.2 Agent 输出示例

```
你好呀~

看到您投递了我们陪诊员的岗位，想简单和您了解一下情况。

请问您是有护士证吗？
continue
```

或最终判断：

```
好的，我们这边非常欢迎有护士证的陪诊员加入，期待和您进一步沟通！
true
```

脚本自动将末尾标签（`true` / `false` / `continue`）识别并去除，仅发送上方正文部分。

## 十、注意事项

1. **Dify 部署**：脚本默认连接内网地址 `http://192.168.31.20:6080`，如 Dify 部署地址变更需同步修改脚本中 `DifyAgent.BASE_URL`
2. **勿重复运行**：同一 Boss 直聘标签页只应运行一个脚本实例
3. **礼貌用语**：AI 生成消息代表公司形象，建议在 Dify Agent 侧配置好专业、友好的话术风格
4. **候选人体验**：自动沟通强度建议不超过每日招呼 50 人，避免触发平台风控
5. **数据备份**：定期通过「导出数据」按钮导出候选人名单，避免浏览器缓存丢失
